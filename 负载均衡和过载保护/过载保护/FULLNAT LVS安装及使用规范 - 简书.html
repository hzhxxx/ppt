FULLNAT LVS安装及使用规范

作者 a580d7adc2a6 关注
2017.02.24 10:13 字数 1333 阅读 27评论 0喜欢 1

编译及安装

一、Fullnat版本

  1. 下载https://github.com/alibaba/LVS
  2. 版本：V2

二、安装7zip

yum install -y p7zip.x86_64

三、编译过程

  1. root用户下，解压LVS-lvs_v2.zip

    7za x LVS-lvs_v2.zip

  2. 软链接至linux编译目录

    ln -sv /root/LVS-lvs_v2/kernel/ /usr/src/linux

  3. 进入目录编译

    cd /usr/src/linux
    更改.config文件

    CONFIG_IP_VS_TAB_BITS=22
    CONFIG_IP_VS_RR=m
    CONFIG_IP_VS_WRR=m
    CONFIG_IP_VS_LC=m
    CONFIG_IP_VS_WLC=m
    CONFIG_IP_VS_LBLC=m
    CONFIG_IP_VS_LBLCR=m
    CONFIG_IP_VS_DH=m
    CONFIG_IP_VS_SH=m
    CONFIG_IP_VS_SED=m
    CONFIG_IP_VS_NQ=m
    CONFIG_IP_VS_PROTO_TCP=y
    CONFIG_IP_VS_PROTO_UDP=y
    CONFIG_IP_VS_PROTO_AH_ESP=y
    CONFIG_IP_VS_PROTO_ESP=y
    CONFIG_IP_VS_PROTO_AH=y

  4. 编译模块，根据cpu数量调整-J参数

    make -j24 all

  5. 安装模块

    make modules_install

  6. 安装内核及配置文件等

    make install
    报错处理：
    ERROR: modinfo: could not find module acpi_ipmi
    ERROR: modinfo: could not find module sb_edac
    ERROR: modinfo: could not find module lpc_ich
    ERROR: modinfo: could not find module ptp
    解决方法：

    cp /lib/modules/2.6.32-504.el6.x86_64/kernel/drivers/acpi/acpi_ipmi.ko /lib/modules/2.6.32/kernel/drivers/acpi/
    cp /lib/modules/2.6.32-504.el6.x86_64/kernel/drivers/edac/sb_edac.ko /lib/modules/2.6.32/kernel/drivers/edac/
    cp /lib/modules/2.6.32-504.el6.x86_64/kernel/drivers/mfd/lpc_ich.ko /lib/modules/2.6.32/kernel/drivers/mfd/
    mkdir /lib/modules/2.6.32/kernel/drivers/ptp
    cp /lib/modules/2.6.32-504.el6.x86_64/kernel/drivers/ptp/ptp.ko /lib/modules/2.6.32/kernel/drivers/ptp/
    cp /lib/modules/2.6.32-504.el6.x86_64/kernel/drivers/scsi/sd_mod.ko /lib/modules/2.6.32/kernel/drivers/scsi/sd_mod.ko
    cp /lib/modules/2.6.32-504.el6.x86_64/kernel/lib/crc-t10dif.ko /lib/modules/2.6.32/kernel/lib/crc-t10dif.ko
    重新执行make install

  7. 更改启动项

    vi /boot/grub/grub.conf
    更改为default为0

四、调整内核参数

允许送到队列的数据包的最大数目

net.core.netdev_max_backlog = 500000

开启路由转发功能

net.ipv4.ip_forward = 1
net.ipv4.conf.all.arp_ignore = 1
net.ipv4.conf.all.arp_announce = 2

修改文件描述符

fs.nr_open = 5242880
fs.file-max = 4194304
sed -i 's/1024/4194304/g' /etc/security/limits.conf

五、重启生效

reboot

六、编译keepalived、ipvsadm、quagga软件

安装依赖包：

yum -y install popt-devel libnl-devel

1、编译keepalived:

cd /root/LVS-lvs_v2/tools/keepalived

分别执行

./configure、make && make install

创建软链接：

ln -s /usr/local/sbin/keepalived /usr/sbin/

2、编译ipvsadm:

cd /root/LVS-lvs_v2/tools/ipvsadm

执行

make && make install

3、编译quagga:
添加quagga用户：

useradd quagga
cd /root/LVS-lvs_v2/tools/quagga

执行

./configure --prefix=/data/quagga --localstatedir=/var/run/quagga --enable-user='quagga' --enable-group='quagga' --enable-vty-group='quagga'

再执行：

make && make install

创建quagga所需文件夹：

mkdir -p /var/run/quagga
mkdir -p /data/log/quagga
chown quagga /data/log/quagga /var/run/quagga -R

创建软链接：

ln -s /data/quagga/sbin/ospfd /usr/sbin/
ln -s /data/quagga/sbin/zebra /usr/sbin/

七、zebra启动

切换到keepalived-vip目录

cd keepalived-vip
cp quagga /etc/sysconfig/
cp init.d/zebra /etc/init.d/
cp init/zebra.conf /etc/init/
cp zebra.conf /data/quagga/etc/zebra.conf
/etc/init.d/zebra start

注：zebra应在ospfd之前启动。

八、keepalived-vip配置及安装

mkdir -p /etc/keepalived
根据需要修改ospf.tmpl的hostname和area字段，hostname字段要唯一，如

hostname NS-LVS-TEST-248-66
area 0.0.0.1

将keepalived.tmpl、ospfd.tmpl放置到“/”目录

cp keepalived.tmpl ospfd.tmpl /

将kube-keepalived-vip、kube-keepalived-vip.sh、healthcheck.sh放置到/sbin/；

cp kube-keepalived-vip kube-keepalived-vip.sh healthcheck.sh /sbin

将keepalived-vip放置到/etc/init.d；

cp init.d/keepalived-vip /etc/init.d/keepalived-vip
cp init/keepalived-vip /etc/init/

编辑keepalived-vip的配置文件keepalived-vip-conf

CONFIGMAP="--services-configmap=default/vip-configmap-external"
SERVER="--server=100.30.22.4:8080"
USESERVICE="--use-kubernetes-cluster-service=false"
USEADDRESSES="--use-local-addresses=101.13.20.10-50"
USESERVICEPORT="--use-service-port=false"
USELINKADDRESS="--use-link-address=8.8.8.56"

均为kube-keepalived-vip的启动参数；CONFIGMAP是k8s集群中的configmap名，SERVER是k8s master地址，USEADDRESSES是fullnat LVS的local ip地址，local ip地址的生成有localip.sh来完成，将localip.sh复制到/etc/rc.d/init.d/localip.sh，其部分内容如下：

!/bin/bash

arg=$1
dev=eth1
network="10.3.250"
seq="10 50"

需要根据服务器情况进行配置，比如该服务器的localip地址在哪个段上，10 50是代表地址段，如192.168.0.10-192.168.0.50。

USESERVICE表示是否使用k8s的cluster ip创建客户端， USESERVICEPORT表示是否使用k8s的service端口作为负载均衡的端口，当为false时，使用该service暴露的NodePort端口,否则是service的端口，USELIKADDRESS是ospf的互联地址，通常情况下为eth0的地址。

具体而言，针对external LVS配置文件如下，其中相应地址需要因服务器做特殊配置：

CONFIGMAP="--services-configmap=default/vip-configmap-external"
SERVER="--server=100.30.22.4:8080"
USESERVICE="--use-kubernetes-cluster-service=false"
USEADDRESSES="--use-local-addresses=101.13.20.10-50"
USESERVICEPORT="--use-service-port=true"
USELINKADDRESS="--use-link-address=8.8.8.56"

针对internal LVS配置文件如下：

CONFIGMAP="--services-configmap=default/vip-configmap-internal"
SERVER="--server=100.30.22.4:8080"
USESERVICE="--use-kubernetes-cluster-service=false"
USEADDRESSES="--use-local-addresses=101.13.20.10-50"
USESERVICEPORT="--use-service-port=false"
USELINKADDRESS="--use-link-address=8.8.8.56"

将配置文件keepalived-vip-conf放置
到/etc/sysconfig/，并命名keepalived-vip;

cp keepalived-vip-conf /etc/sysconfig/keepalived-vip

修改/etc/rsyslog.conf 添加
local0.* /var/log/keepalived.log

service rsyslog restart

执行/etc/init.d/keepalived-vip start 并检测ospf和keepalived进程是否启动。keepalived-vip的日志位置为/data/log/keepalived-vip.log，ospfd的日志为/data/log/quagga/ospfd.log，zebra的日志为/data/log/quagga/zebra.log
